# Cursor 하네스 엔지니어링

이 문서는 **Cursor(IDE·에이전트·규칙)** 위에서 작업할 때의 하네스다. [Andrej Karpathy가 정리한 LLM 코딩 함정과 대응 원칙](https://x.com/karpathy/status/2015883857489522876)을, 이 저장소의 파일 배치와 사용 흐름에 맞게 옮긴 것이다.

**트레이드오프:** 속도보다 신중함. 사소한 수정은 판단으로 완화한다.

---

## 1. 규칙 층위 (무엇을 어디에 둘지)

| 층위 | 파일 | 역할 |
|------|------|------|
| 항시 주입 | `.cursorrules` | 짧은 경계·검증 지시. 에이전트가 매 세션마다 읽는 얇은 하네스. |
| 상세 지침 | `CLAUDE.md` | 네 원칙의 전문·예시·계획 템플릿. 프로젝트 지침과 **병합**해 해석한다. |
| Cursor 운영 | `CURSOR.md` (본 문서) | Cursor에서 사람·에이전트가 같이 지킬 **운영 규약**과 체크리스트. |

중복 서술을 피한다. 원칙의 “정본”은 `CLAUDE.md`, “항시 제약”은 `.cursorrules`, “이 도구에서의 쓰임”은 본 문서로 나눈다.

## 2. 카파시 의도 → Cursor 관행

### 2.1 구현 전 사고 (Think Before Coding)

- **컨텍스트를 열기 전에** 요구를 한 문장으로 고정할 수 있는지 확인한다. 모호하면 채팅/에이전트에 질문을 먼저 보낸다.
- `@`로 파일·폴더를 붙일 때는 **가정을 줄이는 방향**으로만 붙인다. 전 저장소를 습관적으로 붙이지 않는다.
- 해석이 갈리면 에이전트에게 “한 가지로 구현”을 시키기 전에, 사람이 후보를 정하거나 에이전트에게 후보 나열을 요청한다.

### 2.2 단순성 우선 (Simplicity First)

- Composer·에이전트 지시는 **한 번에 한 목표**를 기본으로 한다. “리팩터링 겸 기능 추가 겸 문서”처럼 묶지 않는다.
- 스캐폴딩·설정·추상화는 **요청에 적힌 범위** 안에서만 만든다. “나중을 위해”는 카파시가 말한 전형적인 과설계다.

### 2.3 정밀한 수정 (Surgical Changes)

- 변경 범위는 diff로 설명 가능해야 한다. **요청과 무관한 포맷·주석·리팩터**는 에이전트에게 시키지 않는다.
- 에이전트가 만든 diff는 **“이 줄이 왜 이 요청인가?”**로 스스로 검문한다. 연결이 약하면 되돌리거나 지시를 좁힌다.
- 무관한 데드 코드는 삭제 지시 없이 **에이전트가 임의 삭제**하지 않게 한다(발견 시 보고만).

### 2.4 목표 중심 실행 (Goal-Driven Execution)

- 작업 시작 시 **검증 가능한 완료 조건**을 프롬프트에 넣는다. 예: “다음 테스트가 통과하면 끝”, “이 URL에서 이 화면만 확인”.
- 단계가 여러 개면 `CLAUDE.md`와 같이 **단계 → 검증**을 적어 에이전트가 같은 루프를 돌게 한다.
- “작동만 하게”로 끝내지 않는다. 재현 방법이나 테스트·수동 확인 단계 중 하나는 명시한다.

---

## 3. 프롬프트·작업 설계 체크리스트

작업을 줄 때 아래를 채우거나, 에이전트에게 채우게 한다.

1. **목표:** 무엇이 바뀌면 성공인가 (한 문장).
2. **비목표:** 이번에 하지 않을 것 (한 줄 이상 권장).
3. **검증:** 테스트·빌드·수동 확인 중 무엇으로 끝낼지.
4. **컨텍스트:** `@`로 붙일 최소 파일·폴더만.

다단계일 때 예시 형식:

```text
1. [단계] → 검증: [확인 사항]
2. [단계] → 검증: [확인 사항]
```

---

## 4. `.cursor/rules`를 쓸 때

프로젝트별 규칙을 디렉터리로 쪼개 쓰는 경우, 내용은 여전히 위 네 원칙과 충돌하지 않게 둔다. “항상 적용” 규칙이 많아지면 하네스가 무거워져 모델이 핵심을 놓치기 쉽다. **짧고 충돌 없는 규칙**을 선호한다.

---

## 5. 요약

- **가정·혼란·트레이드오프**는 숨기지 않는다.
- **최소 코드·최소 diff·최소 컨텍스트**다.
- **완료는 검증으로 말한다.**

원칙 전문은 `CLAUDE.md`, 에이전트 상시 제약은 `.cursorrules`를 본다.
